# Design Thinking — Testing & Iteration

## Prototype Presentation
The prototype was presented to 2 users on Friday(27.02.2026).
The presentation included a search page, results page, 
and story view.

## User Feedback Summary

### Positive
- Bookshelf idea is fun and interactive
- Semantic search concept is well received
- Pop-up on hover idea is interesting
- Random bookshelf feels engaging

### Issues Raised
- Unclear what is being searched (books? stories? short stories?)
- Confused about where books pop up from
- Red color not liked — users prefer sepia tones
- Layout feels overwhelming (bookshelf + list + search bar together)
- No category filtering available
- Unclear if same search always gives same results

### New Ideas from Users
- Bookshelf displays all 10 stories in random positions
- Positions shuffle on each page refresh
- Books light up or stand out if they match the search results
- Pin search bar so it is always visible while scrolling
- Reduce sentence spacing in story popup
- Story view as popup overlay or side by side
- On mobile: hide bookshelf or move it below search bar

## Decisions Made Based on Feedback

| Feedback | Decision | Reason |
|---|---|---|
| Unclear what is being searched | Add clear label "Search short stories" | Reduces confusion |
| Red color | Use sepia/warm tones | User preference |
| Overwhelming layout | Separate bookshelf and results clearly | Better UX |
| Books light up on match | Add to frontend backlog | Fun and useful feature |
| Pin search bar | Add to frontend backlog | Improves usability |
| Category filtering | Add to backlog as optional feature | Good idea for later |
| Mobile layout | Add to frontend backlog | Responsiveness required |
| Random bookshelf | Add to frontend backlog | Engaging feature |

## What We Would Do Differently
- Present clearer labels and descriptions earlier
- Test on mobile screen earlier in the process
- Define the publishing company context more clearly for users
