# Design THinking - Ideation

## User Flow

- USer lands on a search page
- User enters a natural language search phrase
- User clicks "Discover" or "Find" (or presses Enter)
- System shows a loading indicator while results are being fetched
- System shows a list of relevant stories
- User can click a story to open full story
- User can navigate back to search results
- User sees "No stories found" messgae if nothing matches

## Search Page Ideas

- Centered search input field
- Clear placeholder text (e.g., "Search by theme, emotion, or situation...")
- Simple button next to input
- MInimal distraction on page

## Search Results Ideas

- Results displayed as cards or list items
- Show maximum 6 results on page
- If more results exist, show "Load more" button
- Keep layout clean and not overwhelming
- Each result shows:
    - Title
    - Author
    - Publication year
    - Genre
    - Short automatically generated summary (1-2 sentences)
    - Match strength label (e.g "Strong match", "Good match", " Partial match)
- Match strength uses color coding: gteen yellow, orange
- Results ordered by relevance only
- Clear visual separation between results

## Summary Display Ideas

- Summary limited to 1-2 sentences
- Summary should reflect meaning, not copy the first lines
- Summary should help user decide whether to open the story

## Optional Ideas

- Show "No stories found" message if no results
- Possibly allow filtering be genre later
